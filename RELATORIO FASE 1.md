# Relatório Individual

## Joshua

### 1ª Semana – Análise de Quota

Durante a primeira semana, foi realizada a análise de quota necessária para o desenvolvimento do projeto, tendo em consideração os recursos técnicos disponíveis e as limitações operacionais. Esta análise incluiu:

- Avaliação do volume esperado de dados provenientes dos sensores
- Estimativa de armazenamento na base de dados
- Definição da capacidade do servidor
- Previsão de escalabilidade futura

O objetivo foi garantir que a infraestrutura escolhida suportasse o crescimento do sistema sem comprometer desempenho, estabilidade ou custos operacionais.

### 2ª Semana – Preparação da Estrutura Técnica do Projeto (Backend e Base de Dados)

Na segunda semana, foi estruturada a base técnica do projeto, com foco na arquitetura do backend e modelação da base de dados. As atividades realizadas incluíram:

- Definição das tecnologias a utilizar
- Organização do código e separação por camadas (controladores, serviços e repositórios)
- Estrutura de pastas do projeto
- Desenho do modelo relacional da base de dados
- Definição de entidades e relacionamentos
- Normalização das tabelas

Esta fase foi essencial para garantir uma base sólida, organizada e preparada para futuras funcionalidades.

## Filomeno

### 1ª Semana – Pesquisa sobre Materiais Necessários para o Projeto

Durante a primeira semana, foi realizada uma pesquisa detalhada sobre os materiais necessários para a implementação do sistema de monitorização. Foram analisados:

- Diferentes tipos de sensores (CO2, temperatura e humidade)
- Microcontroladores compatíveis
- Componentes de prototipagem
- Requisitos energéticos

O objetivo foi selecionar componentes adequados em termos de:

- Custo
- Precisão
- Compatibilidade técnica com a infraestrutura do projeto

### 2ª Semana – Pesquisa e Estruturação sobre a Integração do Arduino com o Servidor

Na segunda semana, foi estudada e estruturada a integração entre o microcontrolador Arduino e o servidor backend. Foram analisados:

- Protocolos de comunicação
- Formatos de envio de dados (como JSON)
- Métodos de transmissão (HTTP ou MQTT)
- Estratégias para garantir fiabilidade e segurança na comunicação

Esta etapa foi essencial para assegurar que os dados captados pelos sensores pudessem ser transmitidos, processados e armazenados corretamente no sistema.
